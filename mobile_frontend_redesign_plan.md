# 移动端前端页面重构设计方案

根据需求，针对 `project1` 的移动端前端页面进行重新设计。本次重构主要精简角色权限，整合机台档案与查询功能，并简化图片上传流程。

## 1. 角色与功能划分

系统仅保留以下两个登录角色：

### 1.1 入库员 (Inbound)
- **基础功能**：注册、登录、退出。
- **业务功能**：
  - **入库**：扫描或手动录入机台流水号进行入库。
  - **查询**：查询机台信息，点击机台列表项可进入机台详情/编辑页面（上传机台图片）。
  - **查看库位大屏**：查看移动端简版的库位总数、占用及使用率统计。

### 1.2 库管 (Warehouse Manager)
- **基础功能**：注册、登录、退出。
- **业务功能**：
  - **机台档案（整合至查询）**：不需要独立的机台档案维护入口，直接通过“查询”功能搜索出机台，点击机台进入编辑页面。
  - **简化图片上传**：在机台编辑页面只需负责上传图片，**去除原有的图片编号逻辑**，支持直接选择图片上传即可。

---

## 2. 页面与路由架构设计

采用基于角色的动态路由与底部导航（Tabbar）渲染。

### 2.1 公共页面
- `/login` (登录/注册页)：保留现有功能，注册时角色选项简化为“入库员”和“库管”。
- `/profile` (我的页)：展示当前登录用户信息及角色，提供“退出登录”按钮。
- `/machine-edit/:id` (机台编辑/上传页)：从查询列表点击进入，展示机台基础信息并提供简单的多图上传组件 (`van-uploader`)。

### 2.2 底部导航分配 (Tabbar)
- **入库员看到的 Tab**：
  1. 入库 (`/inbound`)
  2. 查询 (`/query`)
  3. 库位大屏 (`/dashboard`)
  4. 我的 (`/profile`)
- **库管看到的 Tab**：
  1. 查询 (档案管理) (`/query`)
  2. 我的 (`/profile`)

---

## 3. 核心功能改造点

### 3.1 “机台编辑”整合入“查询”功能
- **修改 `InventoryQuery.vue`**：在列表 `van-cell` 元素上绑定点击事件（`@click="goToEdit(item.id)"`），将用户导航至具体的机台编辑页面。
- **统一入口**：无论入库员还是库管，查找和编辑机台的入口统一为“查询页”。

### 3.2 简化机台图片上传
- **新建/修改 `MachineEdit.vue`**：
  - 页面顶部展示机台的基础只读信息（机型、流水号、状态等）。
  - 使用 Vant 的 `van-uploader` 组件提供图片直传功能。
  - **去除编号逻辑**：移除原来需要输入或选择图片编号（如“正面”、“侧面”、“局部01”）的表单字段。图片上传后直接与机台 ID 绑定。

### 3.3 权限与路由守卫控制
- 在 `router/index.ts` 中添加全局守卫 `beforeEach`。
- 登录成功后：
  - 入库员默认跳转至 `/inbound`。
  - 库管默认跳转至 `/query`。
- 对未授权页面的访问进行拦截（例如库管访问 `/inbound` 会被重定向）。

## 4. 给 AI IDE 的执行步骤建议

当 AI IDE 读取本方案进行代码实施时，请按照以下步骤处理：

1. **清理角色配置**：修改 `Login.vue`，将注册表单的角色单选框（`van-radio`）限制为 `Inbound`（入库员）和 `Manager`（库管）。
2. **重构路由与导航**：
   - 创建一个包含 `van-tabbar` 的基础布局组件（如 `Layout.vue`）。
   - 根据 `userStore.userInfo.role` 动态计算 `Tabbar` 的展示项。
3. **改造查询列表 (`InventoryQuery.vue`)**：
   - 为列表项添加右侧箭头图标 (`is-link`) 和点击事件。
4. **开发 `MachineEdit.vue` 页面**：
   - 实现无编号要求的纯图片上传界面。
   - 对接现有的文件上传 API，将图片直接关联至选中机台。
5. **清理冗余代码**：删除旧版本中独立且复杂的机台档案编辑页面（若存在）、删除旧的图片编号相关逻辑及组件。
