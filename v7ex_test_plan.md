# V7ex 系统全面自动化测试计划案

## 1. 测试目标与范围
本测试计划旨在为 AI IDE 提供 V7ex 系统的全面功能、集成与边界测试指南。V7ex 是一个全栈式成品管理系统（Vue3 + FastAPI + MySQL），其核心目标是围绕“合同-订单-库存-入库-发货-机台档案”形成完整的业务闭环。

## 2. 测试环境准备
- **前端运行**：`cd frontend && npm run dev` (监听指定端口)
- **后端运行**：`uvicorn api.main:app --reload` (通常监听 `8000` 端口)
- **数据库**：MySQL 实例运行中，执行数据库迁移或初始化脚本。
- **预置数据**：至少需要一个超级管理员（Admin/Boss）账号用于初始化的用户审核和权限分配。

## 3. 角色与权限流转测试
AI 需要模拟不同角色（Sales, Prod/Inbound, Boss/Admin）的独立泳道流程，验证权限隔离：

### 3.1 销售角色 (Sales) 测试路径
- [ ] 注册申请 Sales 角色，等待 Admin 审核。
- [ ] 登录系统，验证路由权限（仅能访问订单、配货等页面，无权访问审核与发货复核）。
- [ ] 在“销售下单”模块创建手工订单。
- [ ] 在“订单配货”模块，选择对应机台并执行配货锁定（库存状态：`待发货`）。
- [ ] 查看订单的发货追踪状态。

### 3.2 生产/入库角色 (Prod / Inbound) 测试路径
- [ ] 注册申请 Inbound 角色，等待 Admin 审核。
- [ ] 登录系统，验证路由权限（侧重入库、档案管理）。
- [ ] 在“成品入库”模块，上传跟踪单文件（测试文件解析或 OCR 功能），或点击“自动生成”。
- [ ] 审核待入库清单，并按具体库位（Location_Code）执行入库操作（库存状态：`库存中`）。
- [ ] 在“机台档案”中上传测试图片/文档，预览并删除。

### 3.3 管理角色 (Boss / Admin) 测试路径
- [ ] 登录 Admin 账号。
- [ ] 在用户管理模块审核上述 Sales 和 Inbound 账号的注册申请。
- [ ] 在“统筹管理”模块，将处于“已规划”状态的合同转为“已下单”并生成订单。
- [ ] 在“发货复核”模块，对已配货的机台执行正式发货（库存状态：`已出库`）。
- [ ] 查看“交易日志”，验证所有关键操作（入库、配货、发货）是否正确记录。

## 4. 核心功能模块测试用例

### 4.1 认证与用户模块 (`/api/v1/auth`, `/api/v1/users`)
- **TC-AUTH-01**: 正常登录，检查响应是否包含 JWT Token 及 User Info。
- **TC-AUTH-02**: 使用错误密码登录，检查是否返回 `401 Unauthorized` 且前端展示错误提示。
- **TC-AUTH-03**: 无 Token 访问受保护接口，验证请求被拦截。

### 4.2 统筹与订单模块 (`/api/v1/planning`)
- **TC-PLAN-01**: 测试获取合同列表与订单列表的数据一致性。
- **TC-PLAN-02**: 对订单执行“配货锁定”，验证目标机台的库存状态流转。
- **TC-PLAN-03**: 释放配货，验证机台状态回滚至 `库存中`，订单占用解除。

### 4.3 库存、入库与发货模块 (`/api/v1/inventory`)
- **TC-INV-01**: 模拟上传跟踪单，验证解析后的暂存数据格式。
- **TC-INV-02**: 确认导入暂存数据，检查对应机台状态变为 `待入库`。
- **TC-INV-03**: 执行按库位入库，验证目标库位容量限制及状态更新。
- **TC-INV-04**: 针对 `待发货` 状态机台，执行发货确认；测试“发货撤回”接口，验证状态是否批量回退到 `待入库`。

### 4.4 档案与日志模块 (`/api/v1/machine-archive`, `/api/v1/logs`)
- **TC-ARC-01**: 指定流水号上传图片附件，调用预览接口确认上传成功。
- **TC-LOG-01**: 检索交易日志，验证特定流水号或订单号的操作轨迹（Traceability）。

## 5. 前后端集成与交互机制测试
- **请求拦截器校验**：抓包验证前端所有的业务请求 Headers 中是否自动携带了 `Authorization: Bearer <token>`。
- **全局异常处理校验**：模拟后端返回 `401` 或 `403`，验证前端路由守卫是否能自动清理本地 Token 并强制跳转至 `/login` 页面。
- **数据加载性能**：测试大型列表（如交易日志或库存全量列表），验证前端分页/虚拟列表及后端的检索性能。

## 6. 异常分支与边界值测试
- **异常测试 1 (配货超量)**：尝试在一个订单中配货超过其需求数量的机台，验证前端预拦截与后端校验报错。
- **异常测试 2 (库位满载)**：向已满载的库位继续添加入库机台，验证后端返回 `400` 级别错误及清晰提示。
- **异常测试 3 (状态机防错)**：尝试直接对状态为 `待入库` 的机台调用“发货确认”接口，验证后端拒绝执行非法状态流转。
